
# AI Trip Planner - "Plan this trip for me"

## Overview
Add an AI-powered trip planning flow to the Plan tab. Users answer a few quick questions, then an LLM generates a complete draft plan (budget breakdown + daily itinerary) that can be saved and edited.

---

## 1. Database Changes

### New table: `trip_plan_sessions`
Stores each planning session's Q&A answers and the generated plan.

| Column | Type | Notes |
|---|---|---|
| id | uuid PK | default gen_random_uuid() |
| trip_id | uuid NOT NULL | |
| user_id | uuid NOT NULL | |
| answers_json | jsonb | User's questionnaire answers |
| generated_plan_json | jsonb | Full LLM output |
| created_at | timestamptz | default now() |

### New table: `trip_plan_items`
Individual plan items (bookings + itinerary) generated by AI or manually added.

| Column | Type | Notes |
|---|---|---|
| id | uuid PK | default gen_random_uuid() |
| trip_id | uuid NOT NULL | |
| category | text NOT NULL | flights, stay, experiences, shared, buffer |
| title | text NOT NULL | |
| description | text | |
| est_cost | numeric | Estimated cost |
| day_number | integer | Nullable, for itinerary items |
| time_block | text | e.g. "morning", "afternoon" |
| source_url | text | Optional link |
| status | text | default 'draft' |
| created_by | uuid | |
| created_at | timestamptz | default now() |

### RLS Policies
- **trip_plan_sessions**: Members can SELECT for their trip; users can INSERT their own sessions
- **trip_plan_items**: Members can SELECT for their trip; members can INSERT with own user_id; organizers can UPDATE (approve drafts); organizers can DELETE

## 2. Edge Function: `generate-trip-plan`

A new backend function that:
1. Receives `trip_id` and `answers_json` (preferences like accommodation style, activity interests, pace, dietary needs)
2. Pulls trip context from DB using service role: destination, dates, total_cost, member count, vibe
3. Calculates trip duration (number of days)
4. Calls the Lovable AI gateway (google/gemini-3-flash-preview) with a structured prompt
5. Uses **tool calling** to enforce JSON schema, returning:
   - `budget_breakdown`: array of {category, amount, notes} summing to total_cost
   - `itinerary_days`: array of {day_number, activities: [{time_block, title, description, est_cost}]}
   - `suggested_options`: array of {category, title, description, est_cost_low, est_cost_high, search_url}
6. Saves session to `trip_plan_sessions`
7. Returns the structured plan

Config: `verify_jwt = false` (manual auth in code, same pattern as create-checkout)

---

## 3. Edge Function: `save-trip-plan`

Receives `trip_id` and `generated_plan_json`, then:
1. Validates user is a trip member
2. Deletes existing `trip_plan_items` with status='draft' for this trip (idempotent regeneration)
3. Inserts new items from the plan JSON into `trip_plan_items` as status='draft'
4. Returns success

---

## 4. UI Changes

### Plan Tab Entry Point
Add a "Plan this trip for me" card at the top of PlanTab (above the booking categories). Shows a sparkle icon and brief description. Clicking opens the AI Planner flow.

### AI Planner Flow (Sheet/Modal)
A bottom sheet (reusing the Sheet component pattern from ReplanChat) with 3 steps:

**Step 1 - Quick Questions** (3-4 questions):
- Accommodation preference: Budget / Mid-range / Luxury / Unique stays
- Activity interests: Adventure / Culture / Food / Relaxation / Nightlife (multi-select)
- Pace: Packed schedule / Balanced / Relaxed
- Any special requests? (free text, optional)

**Step 2 - Generating** (loading state):
- Animated spinner with "Planning your trip..." message
- Non-streaming call (structured output via tool calling)

**Step 3 - Preview & Actions**:
- Budget breakdown cards showing category allocations with "Draft" badges
- Day-by-day itinerary preview
- Suggested options with estimated price ranges and Google search links
- Three action buttons:
  - "Save Draft to Plan" - calls save-trip-plan, closes sheet, refreshes Plan tab
  - "Regenerate" - goes back to loading with same answers
  - "Edit Answers" - goes back to Step 1

### Plan Tab Integration
After saving, show `trip_plan_items` alongside existing bookings/itinerary. Draft items display with a "Draft" badge. Organizers see an "Approve" button to change status from draft to approved.
